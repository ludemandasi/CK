# Contributing to CK

Thanks for your interest in contributing! We welcome bug reports, feature requests and pull requests.

Please follow these steps to make contributing smooth:

1. Fork the repository and create a branch for your change:

```bash
git checkout -b feature/your-feature
```

2. Make small, focused changes with clear commit messages.

3. If you are fixing a bug, include a reproduction case or steps to reproduce.

4. Run tests and linters (if any) before submitting a PR.

5. Open a pull request against the `main` branch with a descriptive title and explanation.

Guidelines

- Keep changes focused and include tests where appropriate.
- Follow existing code style. If no style is defined, follow common conventions for the language used.
- Update documentation (README) when behavior changes.

What to include in a good PR

- A short description of the change and why it's needed.
- The steps to reproduce (for bug fixes).
- Any related issue numbers.
